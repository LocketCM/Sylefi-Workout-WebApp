# Patch Notes

Each release gets its own dated file in this folder. The top-level
[`PATCH_NOTES.md`](../../PATCH_NOTES.md) is the running combined log — copy
new entries there too so Meg only has to look in one place.

## Convention

- File name: `YYYY-MM-DD.md` (e.g. `2026-04-08.md`)
- Group changes under **Coach**, **Client**, and **Under the hood**
- Keep it human — this is for Meg, not a git log
- One file per release day; if you ship twice in a day, append to the same file
